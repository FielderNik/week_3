# DESIGN.md

Документ описывает визуальный стиль `chat_template/`. Референс: [ChatGPT design system в Refero Styles](https://styles.refero.design/style/52a007ed-ad1b-46a6-bd44-b76f91df6d0c).

## Образ

Интерфейс должен ощущаться как спокойная рабочая поверхность для текста: почти монохромная страница, минимум визуального шума, один главный объект внимания - поле ввода сообщения. Функциональные панели допустимы, но они не должны спорить с чатом.

В шаблоне остаются три зоны:

- слева - история сохраненных диалогов;
- по центру - текущий чат и composer;
- справа - настройки AI-провайдера.

## Цвета

Используй нейтральную палитру без брендовых акцентов, градиентов и декоративных цветов.

| Токен | Значение | Роль |
| --- | --- | --- |
| `--color-ink` | `#0d0d0d` | основной текст, иконки, темные кнопки |
| `--color-smoke` | `#5d5d5d` | вторичный текст |
| `--color-ash` | `#8f8f8f` | placeholder, неактивные подписи |
| `--color-fog` | `#ececec` | тонкие границы, hover-состояния |
| `--color-paper` | `#f9f9f9` | фон боковых панелей и страницы |
| `--color-snow` | `#ffffff` | основная рабочая сцена и поля |

Состояния ошибок можно показывать приглушенным красным, но не превращать его в брендовый акцент.

## Типографика

- Основной шрифт: `ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`.
- Размеры: `14px`, `16px`, `18px`, `24px`.
- Максимальный вес: `600`.
- Не используй display-веса `700-900`.
- Для текстового интерфейса включай `font-feature-settings: "liga" 0`.

Роли:

| Роль | Размер | Line-height | Вес |
| --- | --- | --- | --- |
| `caption` | `14px` | `20px` | `400-500` |
| `body` | `16px` | `24px` | `400` |
| `subheading` | `18px` | `28px` | `500-600` |
| `heading` | `24px` | `32px` | `400-500` |

## Радиусы и расстояния

- Интерактивные элементы: `10px`.
- Крупные контейнеры: до `28px`, только если это действительно большой блок.
- Базовые промежутки: `6px`, `8px`, `10px`, `12px`, `16px`, `20px`.
- Между крупными зонами держи `20-32px`, без декоративных разделителей.

## Компоненты

### App shell

Desktop: трехколоночная сетка `260px / 1fr / 320px`. Боковые панели живут на фоне `Paper`, основная сцена - `Snow`. Разделение строится на разнице поверхностей, а не на тенях.

Mobile: одна колонка. История и настройки идут компактными блоками вокруг чата. Горизонтального overflow быть не должно.

### Боковая история

Панель без теней. Заголовок компактный, элементы истории - как nav items:

- радиус `10px`;
- hover/active через фон `Fog`;
- иконки line-style, `1.5px`;
- текст `14px`.

### Основная сцена

Header минимальный: название шаблона и действия `Сохранить` / `Очистить`. Empty state содержит один спокойный вопрос и короткую техническую строку, без иллюстраций и карточек.

### Сообщения

Сообщения не должны выглядеть как цветные bubbles. Сообщение пользователя можно слегка отделить `Fog`-фоном, ответ ассистента остается на белой поверхности. Markdown-контент читаемый, с тонкими границами у code blocks и таблиц.

### Composer

Главный визуальный объект:

- белый фон;
- граница `1px solid --color-fog`;
- радиус `10px`;
- внутри textarea и темная кнопка отправки;
- без тени и без цветного focus glow.

### Правая панель настроек

Это utility-зона, а не отдельный dashboard. Сохраняй плотную форму, маленькие labels и спокойные поля. `details`/`summary` допустимы, если позже потребуется сворачивать группы.

## Do

- Используй один темный токен `#0d0d0d` для текста, иконок, границ и filled controls.
- Разделяй зоны через `Paper` и `Snow`, а не через тени.
- Держи UI компактным и текстовым.
- Делай focus-visible заметным, но монохромным.
- Проверяй desktop и mobile после визуальных правок.

## Don't

- Не добавляй цветные CTA, gradients, glow, декоративные иллюстрации или marketing sections.
- Не используй pill radius `999px`.
- Не добавляй heavy shadows.
- Не делай вложенные карточки.
- Не размазывай provider-логику по UI: API-запросы остаются в provider-модулях.
